About this Project

This is another electronic version of the book The Hacker Crackdown by Bruce Sterling. It has two main aims:

  • To be easily readable in a web browser, regardless of device
  • To be openly annotatable. By openly I mean by anyone without any form of identification.
  • For those annotations to be visible to anyone.
How does it work?

Highlight any text (yes including on this page). Click on the pencil. Type your comment. Click Save. That's it.

A few more detailed notes:

  • If you want to be able to edit or remove your annotation at a later point, then also enter a username beneath your comment. It can be anything you like (as long as somebody else hasn't taken it).
  • If you don't enter a username your annotation will show up as the user "anon". You will NOT be able to delete or modify annotations made in this way.
  • Annotations are moderated. At first you will see your own annotations in blue. Once they're moderated they will turn yellow. While they're blue no-one else can see them. Note: this behaviour uses local storage in the browser. If you don't have this, for whatever reason, then nothing will show until it is moderated.
  • Moderation will focus on weeding out spam, gibberish and horrible shit. I will try to be as open as possible. If you think something has been miscategorised, then email me: scandox at gmail.
  • If you change username, then you will lose the ability to edit and remove prior annotations (unless you copy your token from localStorage manually and restore it - see below)
Some technical matters

The site aims to gather as little data about users as possible, while still being useful.

  • The site doesn't use Cookies at all.
  • I don't use any externally linked resources, CDNs etc
  • The site has no concept of `password`
  • The server running this version (hc.selectedintelligence.com) has all the logging I know about disabled
  • The site DOES use localStorage extensively. If you choose to pick a username when making annotations the server generates a JWT. It also uses localStorage to keep track of your unmoderated annotations.
  • If you have chosen a username, the JWT stored in localStorage will be added to the X-Access-Header with each request to the server. The JWT encodes only the username, issuing server and the issue date. The JWT is not stored server-side. It relies only on the decoding with the server-side secret to establish the claim to the username. They cannot be revoked individually.
  • Your username will NOT be available to you on other devices, unless you manually copied over the user entry from localStorage to the other device's localStorage for the site.
  • If you change username for a specific annotation, then the localStorage entry is overwritten and you won't then be able to edit and remove annotations made with the previous username. However you could, in theory, copy, retain and restore a prior user entry to achieve a multi-user situation. Probably it would be a pain in the neck though.
Why? and other matters

Why did you do this? I think this book's relevance has in many ways increased over time, rather than decreased. I'd like more non-technical people to be aware of it. I thought a lot of people probably had very interesting knowledge of and reflections upon the events described.

Where's the search function? There is no search function. The site is designed as a reading experience. A start at the beginning and end at the end type experience. If you want to search the annotations, you can download the raw data and hack away.
